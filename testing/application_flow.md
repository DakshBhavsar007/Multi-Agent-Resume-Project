## 5. 📄 Resume Upload & ATS Parsing Flow

```mermaid
flowchart TD
    A([Candidate uploads\nresume]) --> B[Select File\nPDF or DOCX]

    B --> C{Client-side\nValidation}
    C -->|Wrong format| D[Error: Only PDF/DOCX supported]
    C -->|File too large| E[Error: Max size is 10MB]
    C -->|Valid| F[POST /api/v1/seeker/resume/upload]

    D --> B
    E --> B

    F --> G[Django REST parser]
    G --> H{Server-side\nFile Validation}
    H -->|Invalid| I[400 Error: Format rejected]
    H -->|Valid| J[Save file locally\nuploads/seekers/seekerId/uuid.pdf]

    J --> K[Celery async task trigger\nparse_resume_job]
    K --> L[Candidate sees:\nAI Ingesting details...]

    L --> M[Resume Parser Agent\nExtracts raw text via PyMuPDF]
    M --> N[NLP Extraction Pipeline\nIdentify sections: Experience, Skills, Education]
    
    N --> O[Skill Normalization Agent\nMap raw skills to 1000+ standard taxonomy]
    O --> P[ATS Compatibility Agent\nCalculates ATS Report and Scores]
    P --> Q[Auto-sync details to seeker profile]
    Q --> R[Dashboard updated with parsed data]
```

## 6. 🎙️ AI Evaluation & Interview Flow

```mermaid
sequenceDiagram
    actor Candidate
    participant FE as React Frontend
    participant DJ as Django Backend
    participant AG as AI Interview Agent
    participant DB as Supabase DB

    Candidate->>FE: Click: Start Evaluation Round
    FE->>DJ: GET /api/v1/sessions/{session_id}/rounds

    alt MCQ Round
        DJ->>AG: Generate custom technical MCQs matching JD
        AG-->>DJ: Return MCQs list
        DJ-->>FE: Load MCQ interface
        Candidate->>FE: Submit answers
        FE->>DJ: POST /api/v1/rounds/{round_id}/submit-mcq
        Note over DJ: Auto-grade and save score to DB
    else Coding Round
        DJ->>AG: Generate coding problems (Python/JS)
        AG-->>DJ: Return challenges
        DJ-->>FE: Render custom code editor
        Candidate->>FE: Write & Run Code against test cases
        Candidate->>FE: Submit final code
        FE->>DJ: POST /api/v1/rounds/{round_id}/submit-code
        Note over DJ: Save coding score to DB
    else AI Voice/Chat Mock Interview
        FE->>DJ: GET /api/v1/rounds/{round_id}/interview/next-question
        DJ->>AG: Generate question based on resume + JD
        AG-->>DJ: Question text
        DJ-->>FE: Speak question (TTS) or show text
        Candidate->>FE: Answer via audio (STT) or chat input
        FE->>DJ: POST /api/v1/rounds/{round_id}/interview/submit-answer
        Note over DJ: Evaluate answer & store response
        Note over FE,DJ: Loop for 5 conversational questions
        FE->>DJ: POST /api/v1/rounds/{round_id}/interview/complete
        DJ->>AG: Generate final scorecard & feedback report
        AG-->>DJ: Scorecard JSON
        DJ->>DB: Save interview results
        DJ-->>FE: Render final interview performance report
    end
```

## 7. 🔍 Fraud Score (Trust Score) Flow

```mermaid
flowchart TD
    A([Recruiter posts job\nor Seeker scans URL]) --> B[Scan request submitted]
    B --> C[Fraud Detection Agent triggered]

    C --> D1{Website Domain\nreachable?}
    C --> D2{Corporate domain\nmatches email?}
    C --> D3{Salary ranges\nrealistic vs market?}
    C --> D4{LinkedIn page\nexists?}
    C --> D5{Boilerplate scan\ncloned text?}

    D1 -->|No| P1[Penalty: site offline]
    D2 -->|No| P2[Penalty: public email domain]
    D3 -->|No| P3[Penalty: scam salary]
    D4 -->|No| P4[Penalty: ghost presence]
    D5 -->|Yes| P5[Penalty: copied content]

    P1 & P2 & P3 & P4 & P5 --> CALC[Compute Trust Score /100\nAssign Risk level: Low/Med/High]
    CALC --> DB[Save scan history to DB]
    DB --> FE[Render detailed interactive safety metrics]
```

## 8. 🔐 Authentication & Token Blacklisting Flow

```mermaid
flowchart TD
    A([User clicks login/logout]) --> B{Action?}

    B -->|Login| C[Submit Credentials]
    C --> D{Verify Password hash}
    D -->|Invalid| E[Increment Redis IP attempts]
    E --> F{IP limits exceeded?}
    F -->|Yes| G[Lock IP in Redis for 15 min]
    F -->|No| H[Error: Invalid login]

    D -->|Valid| I[Generate JWT access + refresh tokens]
    I --> J[Store active session]

    B -->|Logout| K[POST /api/v1/auth/logout]
    K --> L[Extract Bearer token from header]
    L --> M[Save token to Redis blacklist\nblacklist:token with TTL 7 days]
    M --> N[Confirm logout to client]

    N --> O{User requests resource?}
    O --> P[Auth decorator inspects token]
    P --> Q{Is token in Redis blacklist?}
    Q -->|Yes| R[Return 401: Logged Out / Unauthorized]
    Q -->|No| S[Validate signature and grant access]
```
